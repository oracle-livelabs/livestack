# Scene 10 Governed Agent Console

## Introduction

**Governed Agent Console** closes the journey with auditable AI-assisted action. The operational event is a regulated supply question that may require routing to quality-signal analysis, cold-chain service routing, or clinical supply exposure logic. The business risk is that AI recommendations become untraceable if teams cannot see what was asked, which tool path was used, what data was returned, and whether the action was logged.

In life sciences, speed only helps if control is preserved. A sponsor, CRO, clinical supply team, quality organization, or manufacturing partner needs the assistant to stay tied to governed data, visible tool use, reviewable evidence, and durable audit activity.

The page helps the user decide whether an AI-assisted recommendation is ready for review, escalation, or operational follow-up. Oracle AI Database supports the pattern by keeping SQL and PL/SQL tools close to governed operational data while the app records auditable activity for later review.

Estimated Time: **10 minutes**

### Objectives

In this scene, you will learn how AI-assisted workflows can remain auditable, what evidence the user should inspect, and how a regulated supply question can move to reviewable action.

## Task 1: Review the agent console workspace

![Governed Agent Console overview with runtime profile, example questions, and recent actions highlighted](images/agent-console.png)

Perform the following set of steps to review how AI-assisted operations can remain tied to governed data and audit evidence.

1. Click **Governed Agent Console** in the sidebar.
2. Review the runtime profile selector in the top right.
3. Review the example questions in the chat panel.
4. Review **Recent Agent Actions** below the chat panel.

The page frames governed AI as an operational assistant, not an untraceable chatbot. The expected action is not automatic execution; it is reviewable support for a business decision.

## Task 2: Run the clinical supply exposure agent question

![Agent response with clinical supply exposure table and tool badges highlighted](images/agent-clinical-supply-response.png)

Perform the following set of steps to run a question that connects natural language, agent routing, SQL tooling, and returned clinical supply data.

1. Click **Ask** on **Show clinical supply exposure by regulated product category**.
2. Review the agent response at the top of the chat output.
3. Review the clinical supply exposure table.
4. Review the tool badges below the response.

In the current demo dataset, the agent summarizes the last **30** days: **1,514** orders, **\$40,329,169** revenue, **31.0%** signal-driven exposure, and **1,066** unique trial sites. The table shows **Cell and Gene Therapy** with **221** orders and **\$13,499,200** in revenue.

The business decision is whether this exposure pattern should drive supply planning, quality review, or leadership escalation.

## Task 3: Interpret the operational story

![Agent response with clinical supply exposure data highlighted](images/agent-clinical-supply-response.png)

Perform the following set of steps to review why governed agent workflows matter in regulated operations.

1. The question narrows the analysis to clinical supply exposure by regulated product category.
2. Oracle data identifies category-level orders and revenue.
3. The agent summarizes exposure and displays the supporting rows.
4. The business user can compare whether the exposure pattern aligns with the quality signals, cold-chain coverage, and model-backed risk views from earlier scenes.

This is the operational point of the scene: the assistant is useful because it returns governed evidence and keeps the tool path visible. A team can use the response to decide whether to initiate a review, adjust supply planning, contact a site or depot, or document a follow-up action.

## Task 4: Review the agent action audit trail

![Recent Agent Actions audit trail highlighted](images/agent-action-audit-trail.png)

Perform the following set of steps to review the audit trail and confirm how the demo supports traceability.

1. Scroll to **Recent Agent Actions**.
2. Review the top action row.
3. Confirm that the row records the **chat query**, routing details, and supporting action metadata.
4. Review the confidence value.

This is the compliance moment in the scene. Regulated organizations can use AI-assisted workflows only when the actions, tool paths, and decision evidence are reviewable. The takeaway is speed with control: the AI-assisted path helps the user move faster, while the evidence and audit activity remain visible.
